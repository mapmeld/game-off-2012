The Rules are simple

You run the Twitter Bootstrap GitHub repo (an online code library).

You face several incoming pull requests (code changes).

You gain a point for shooting down "bad", rejected pull requests or letting "good", accepted ones pass through.

You lose a point for doing it the other way around.

Use the pull request message to determine friend from foe.


Based on Andy Mason's JavaScript Missile Command
